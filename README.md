# Dumpspace Viewer

A modern desktop application for viewing, searching, and analyzing Unreal Engine dump data generated by [Dumper-7](https://github.com/Encryqed/Dumper-7). This tool provides an intuitive interface for exploring classes, structs, functions, enums, and offsets from Unreal Engine game dumps, with powerful search capabilities, inheritance visualization, and code generation features.

## What is This Project?

Dumpspace Viewer is designed to help developers and reverse engineers work with Unreal Engine dump data more efficiently. It takes JSON dump files generated by Dumper-7 and provides:

- **Fast Search**: Instantly search through thousands of classes, structs, functions, enums, and offsets
- **Inheritance Visualization**: Visualize and navigate class inheritance hierarchies
- **Code Generation**: Generate C++ and C# offset code for your projects
- **Collection Management**: Organize and manage offsets and members into collections
- **Advanced Member Search**: Search for members across inheritance chains and referenced types
- **MCP Server**: Query dumps programmatically from AI assistants (Claude Code / Codex) over the Model Context Protocol

Useful for modding, reverse engineering, and Unreal Engine development workflows.

## Features

- **Multi-type Search**: Search across Classes, Structs, Functions, Enums, and Offsets simultaneously
- **Type Search Mode**: Search by member type instead of member name
- **Inheritance Visualization**: View complete inheritance chains with clickable navigation
- **Inheritance Graph** (Beta): Interactive visual graph of class inheritance relationships
- **Code Generation**: Generate C++ and C# struct/offset code with proper formatting
- **Collection System**: Create and manage collections of offsets and members
- **Advanced Member Search**: Search members across parent/child classes and referenced types
- **Real-time Search**: Instant results as you type
- **Modern UI**: Dark theme with responsive design
- **File-based Storage**: Collections saved to file system

## Installation & Running

### Prerequisites

- **Node.js** 20.9.0+ and **npm** (required for Electron 39.2.7)
- **Windows 10/11** (for built executable)

### Development Mode

1. Clone or download this repository

2. Install dependencies:
```bash
npm install
```

3. Run the application:
```bash
npm start
```

### Building for Windows

1. Install dependencies (if not already done):
```bash
npm install
```

2. Build the executable:
```bash
npm run build:win
```

The built application will be in the `dist` folder:
- **Installer**: `dist/Dumpspace Viewer Setup 1.0.0.exe` (NSIS installer)
- **Portable**: `dist/win-unpacked/Dumpspace Viewer.exe` (portable executable)

**Note**: To set a custom icon, place `build/icon.ico` in the project root before building.

## Usage

1. **Prepare Your Dump Files**: 
   - Use [Dumper-7](https://github.com/Encryqed/Dumper-7) to generate dump files from your Unreal Engine game
   - You'll need: `ClassesInfo.json`, `StructsInfo.json`, `FunctionsInfo.json`, `EnumsInfo.json`, `OffsetsInfo.json`

2. **Load Your Dumps**:
   - Launch the application
   - Click "Select Folder" and choose the directory containing your JSON dump files
   - The application will automatically load all JSON files

3. **Search & Explore**:
   - Use the search bar to find classes, structs, functions, enums, or offsets
   - Click on results to view detailed information
   - Navigate inheritance chains using the inheritance path display
   - Use the "Graph View" button to visualize inheritance (Beta feature)

4. **Generate Code**:
   - View classes/structs to see generated C++ code
   - Use the "Copy Code" button to copy to clipboard
   - Or use "Export to File" to save code to a file

5. **Manage Collections**:
   - Right-click on members to add them to collections
   - Create collections to organize offsets for your projects
   - Generate C++ or C# offset code from collections
   - Export/import collections for sharing

## MCP Server (Claude Code / Codex)

In addition to the desktop UI, this project ships a local stdio [MCP](https://modelcontextprotocol.io) server so AI assistants can query dumps directly. See [docs/MCP.md](docs/MCP.md) for full setup.

Run it:
```bash
npm run mcp
```

It exposes these tools:

- `load_dump_folder` — load a folder of Dumper-7 JSON files (required first)
- `get_dump_status` — report the loaded folder and per-kind symbol counts
- `search_symbols` — search classes, structs, functions, enums, and offsets (glob `*`, OR `|`, filters like `kind:` and `inherits:`)
- `search_members` — search a class/struct's members by name or type, optionally including inherited members
- `get_symbol_detail` — expand a symbol: class/struct size and members, enum values, or a function's signature, address, and flags
- `explain_type_relationship` — determine whether one type can be cast to another via inheritance

Register it with Claude Code by pointing an MCP server entry at `node src/mcp/server.mjs`, or with Codex via `.codex/config.toml`.

## Beta Features

### Inheritance Graph (Beta)

The inheritance graph visualization is currently in **beta** and provides an interactive visual representation of class inheritance relationships. Features may be limited or subject to change.

**⚠️ Memory Considerations**: 
- The graph view has been optimized with hard limits (500 nodes, 1000 edges) to prevent excessive memory usage
- When viewing large classes with many members, the graph will automatically cap at these limits to prevent memory issues
- For best performance, consider:
  - Reducing parent/child depth when viewing large classes
  - Disabling "Show Members" for very large classes
  - Disabling "Show Type References" when members are enabled (automatically disabled)
- If you encounter memory warnings, the graph will stop building and display what was successfully generated

**Features**:
- Interactive graph with zoom, pan, and node navigation
- Right-click context menu to navigate to related classes
- Search functionality to filter nodes
- Configurable depth for parent/child relationships
- Optional member and type reference visualization

To access:
1. View a class or struct
2. Click the "Graph View" button in the inheritance path section
3. Explore the interactive graph (zoom, pan, click nodes to navigate)
